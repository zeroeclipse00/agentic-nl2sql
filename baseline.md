#### 长 Prompt 截断策略

NL2SQL 的 prompt 包含完整的数据库 schema，token 长度差异极大（440 ~ 12780 token）。分析 6656 条训练数据的 prompt 长度分布：

| 阈值 | 覆盖率 |
|------|--------|
| ≤4096 | 95.4% |
| 4096~12288 区间 | 0%（无样本） |
| ≤16384 | 100% |

prompt 长度呈两极分布，长 prompt 全部集中在 12000+ 区间。最终选择 `max_prompt_length=8192`（覆盖 95.4%），超出的 4.6% 采用**左截断**（`truncation=left`），保留 prompt 末尾的 Question 和 Hints 部分（这些是生成 SQL 最关键的信息），截掉的是前面部分 schema 表定义。
